Таймзона портала
MOD-TZ-1. Таймзона портала Europe/Moscow
Установить таймзону портала: Europe/Moscow. Она используется для:
“календарных суток” в оценке полезности (сброс в 00:00)
определения “сегодня” в пересмотрах и уведомлениях
расписаний ежедневных проверок и эскалаций
вычисления дат/ограничений, завязанных на календарный день.
5) Оценка полезности (helpful / not helpful)
MOD-RATE-1. Ограничение: 1 оценка на материал в календарный день
Один пользователь может оценить один материал не чаще 1 раза в календарный день.
День определяется по таймзоне портала (Europe/Moscow).
В 00:00 (MSK) ограничение “сбрасывается” автоматически (визуальный сброс не требуется).
ADD-RATE-2. Правило неизменности оценки в течение дня
В течение дня пользователь не может изменить оценку (любая повторная попытка считается повторной оценкой и запрещена).
ADD-RATE-3. Техническое требование (надёжность)
Ограничение должно быть обеспечено на уровне данных (например, уникальность на пару user+material+date), чтобы его нельзя было обойти через API.
6) Просмотры и анти-накрутка
MOD-VIEW-1. Что считается просмотром
Просмотр = открытие карточки материала.
ADD-VIEW-2. Анти-накрутка просмотров (dedup)
Один пользователь увеличивает счётчик просмотров материала не чаще 1 раза в 30 минут.
Дедуп выполняется по user_id (без учёта устройства/браузера).
Значение окна — настраиваемое, дефолт: 30 минут.
При этом события аудита (по желанию) могут фиксировать все открытия карточки, но агрегат просмотров для витрин/популярности растёт только по dedup-правилу.
7) Витрина “Популярное” и “Самые полезные”
MOD-POP-1. “Популярное” рассчитывается как просмотры + полезность
Использовать параметры по умолчанию:
период просмотров: 30 дней
веса: w1=0.7, w2=0.3, w3=0
сглаживание полезности: m=20
просмотры берутся по правилу из раздела 6 (карточка + dedup)
Формула:
helpfulness_score = (helpful + m*C) / (ratings_total + m)
где C — средняя полезность по порталу.
popularity_score = 0.7 * log(views_30d + 1) + 0.3 * helpfulness_score
ADD-POP-2. Витрина “Самые полезные”
Сортировка по helpfulness_score DESC
Порог достоверности: ratings_total >= 5 (min_ratings = 5)
8) Сортировки в списках
ADD-SORT-1. Сортировка на страницах со списками
На всех страницах/виджетах со списками должна быть сортировка (минимум):
по дате обновления/публикации
по популярности
по критичности
по статусу
по дате следующего пересмотра/просрочке (если применимо)
Конкретный набор может отличаться по сущности (материалы/RFC/очередь согласования), но сортировка должна быть везде, где есть список.
