# Функциональные требования: AI-помощник для преобразования инструкций в HTML

## 1. Назначение
При создании материала добавить отдельную функцию в разделе "Контент" отдельную кнопку "Создание материала с помощью  AI", который позволяет автору базы знаний превратить готовую инструкцию (PDF, DOCX или просто текст) в HTML-страницу единого корпоративного формата и тут же на месте дополнить её картинками — без переключения между программами и без ручной вёрстки. Открывается большое всплывающее окно, в котором Автор продолжает работать с использованием этой функции.

## 2. Настройка модели AI
Выполняется в разделе администрирования, отдельно от AI-помощника по поиску, кроме основных настроек подключения AI-моделей должен быть раздел для указания системного промта с правилами оформления инструкций.

## 3. Пользовательский сценарий (сквозной путь)
1. Автор материала открывает инструмент и загружает исходник — файл (PDF/DOCX) или вставляет текст в поле.
2. Нажимает «Сгенерировать» — инструмент вызывает AI с правилами оформления и получает готовый HTML-код.
3. Код появляется в редакторе: слева код, справа живой предпросмотр.
4. На местах, где AI нашёл упоминание скриншота в исходнике, в превью видны плейсхолдеры — автор кликает на них и выбирает файл картинки.
5. Автор материала может вставить картинку и в любом другом месте — кликом по нужному блоку в превью или кнопкой в редакторе кода.
6. Перед публикацией инструмент проверяет страницу по чек-листу и показывает, что не готово.
7. Автор материала нажимает «Экспортировать» — получает готовый самостоятельный HTML-файл.

## 4. Экран и компоненты интерфейса
Один экран, без отдельных страниц и роутинга.

- Блок загрузки исходника — drag&drop зона, кнопка «Выбрать файл» (PDF, DOCX) и текстовое поле для вставки текста напрямую. Кнопка «Сгенерировать» активна, когда есть хотя бы один источник данных.
- Панель состояния генерации — индикатор «AI обрабатывает файл...», при ошибке — сообщение с кнопкой «Повторить».
- Редактор кода (левая половина) — текстовое поле с HTML, доступное для прямого редактирования; позиция курсора используется как точка вставки для кнопки «Вставить картинку».
- Живой предпросмотр (правая половина) — `iframe` с `srcdoc`, обновляется при каждом изменении кода.
- Плейсхолдеры в превью — визуально выделенные блоки (пунктирная рамка), кликабельные; клик открывает выбор файла.
- Тулбар над редактором — кнопки «Вставить картинку», «Проверить перед публикацией», «Скачать резервную копию», «Экспортировать».
- Панель валидации — список проблем с указанием, что нужно исправить; кнопка экспорта неактивна, пока есть блокирующие проблемы.

## 5. Функциональные требования

### 5.1 Приём исходника
- FR-1.1: Принимать файлы PDF и DOCX, а также вставленный текст.
- FR-1.2: Извлекать текст и структуру из DOCX напрямую — заголовки, списки, нумерация по стилям документа.
- FR-1.3: Извлекать текст из PDF; если текстового слоя нет (сканированный документ), прогонять через OCR и помечать результат как менее надёжный — показывать предупреждение перед генерацией.
- FR-1.4: Перед генерацией удалять артефакты извлечения: номера страниц, колонтитулы, разрывы слов на стыке строк.
- FR-1.5: Ограничение на размер исходного файла — не более 20 МБ; при превышении — понятная ошибка.

### 5.2 Генерация HTML через AI
- FR-2.1: По кнопке «Сгенерировать» отправлять нормализованный текст в AI с системным промптом из файла правил оформления (`trebovaniya_instrukcii_html.md`).
- FR-2.2: AI пропускает опциональные разделы (предварительные требования, особые случаи, частые вопросы), если в исходнике для них нет материала, и не придумывает содержимое для обязательного раздела «Пошаговая инструкция» — если шаги не распознаны, оставляет явную текстовую пометку в коде для автора.
- FR-2.3: При упоминании скриншота в исходнике AI вставляет на это место HTML-плейсхолдер с уникальным `id` и коротким описанием контекста (формат — по файлу правил).
- FR-2.4: При ошибке вызова AI (таймаут, сетевая ошибка, невалидный ответ) — показывать ошибку и кнопку «Повторить», не блокируя интерфейс.
- FR-2.5: Результат генерации помещается в редактор кода целиком, без ручного копирования автором.

### 5.3 Редактор кода и live-preview
- FR-3.1: Код и предпросмотр синхронизированы в обе стороны: правка кода обновляет превью, вставка картинки через превью обновляет код.
- FR-3.2: Обновление превью — с задержкой 300–500 мс после последнего изменения, чтобы не пересчитывать рендер на каждое нажатие клавиши.
- FR-3.3: Превью рендерится в изолированном `iframe` (sandbox), без доступа к остальному приложению.
- FR-3.4: Редактор должен поддерживать синтаксис HTML и CSS.

### 5.4 Вставка изображений
- FR-4.1: Клик по плейсхолдеру в превью открывает системный выбор файла и заменяет именно этот плейсхолдер на `<img>` с тем же `id`.
- FR-4.2: Кнопка «Вставить картинку» в тулбаре вставляет новый `<img>`-блок в позицию курсора в коде — для мест, не покрытых плейсхолдерами.
- FR-4.3: Перетаскивание файла картинки на блок в превью вставляет изображение сразу после этого блока.
- FR-4.4: Перед вставкой изображение автоматически сжимается на клиенте через Canvas API: ширина не более 1200px, формат JPEG, качество 75–80% (PNG — если в оригинале есть прозрачность), и кодируется в base64 прямо в `src`.
- FR-4.5: Если сжатое изображение превышает 400 КБ — показать предупреждение, но не блокировать вставку.
- FR-4.6: К каждому вставленному `<img>` автоматически добавляется пустой атрибут `alt`, который автор обязан заполнить — проверяется в чек-листе валидации (см. 5.5).

### 5.5 Валидация перед публикацией
- FR-5.1: По кнопке «Проверить» запускать чек-лист: все `<img>` имеют непустой `alt`; не осталось нерешённых плейсхолдеров; раздел «Пошаговая инструкция» заполнен, без пометок о нераспознанных шагах; суммарный размер HTML-кода не превышает 5 МБ.
- FR-5.2: Невыполненные обязательные пункты (alt, плейсхолдеры, обязательный раздел) блокируют кнопку «Экспортировать»; превышение размера страницы — только предупреждение.
- FR-5.3: Каждая проблема в списке кликабельна и ведёт к соответствующему месту в коде/превью.

### 5.6 Экспорт и публикация
- FR-6.1: Кнопка «Экспортировать» отдаёт готовый HTML двумя способами: копирование в буфер обмена и скачивание `.html`-файла. 
- FR-6.2: Кнопка «Опубликовать» преобразует код в готовую html-страницу материала и запускает стандартные процедуры публикации материала.  

### 5.7 Повторная генерация
- FR-7.1: При повторном запуске генерации (например, после правки исходника) показывать предупреждение: это перезапишет весь текущий код, включая вставленные картинки.
- FR-7.2: Перед повторной генерацией предлагать скачать текущую версию как резервную копию (`.html`-файл).
- FR-7.3: Умное сохранение ранее вставленных картинок при повторной генерации — вне рамок.
