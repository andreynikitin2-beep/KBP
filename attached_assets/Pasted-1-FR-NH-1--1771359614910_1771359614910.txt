1) Термины и флаг в паспорте
FR-NH-1. Флаг “Для новых сотрудников” в паспорте версии
В паспорт версии материала добавить флаг:
«Требуется ознакомление для новых сотрудников» (new_hire_required = true/false)
Кто может менять: Владелец / Заместитель / Администратор (Автор — в черновике при разрешении).
Важно: флаг меняется только через новую версию и вступает в силу после публикации.
FR-NH-2. Эффективное значение на уровне материала
При публикации версии:
material.new_hire_required = published_version.new_hire_required
2) Админка: раздел “Новые сотрудники” + включатель
FR-NH-3. Раздел “Новые сотрудники”
В админке добавить раздел «Новые сотрудники».
FR-NH-4. Ползунок включения функционала
В разделе есть переключатель (feature flag):
«Включить управление новыми сотрудниками» (new_hires_enabled)
Если выключен:
раздел может быть скрыт/неактивен,
пользователи не добавляются в “очередь новых сотрудников”,
рассылка заданий из этого раздела недоступна.
3) Кто попадает в “Новые сотрудники”
FR-NH-5. Автопопадание новых пользователей
Если функционал включен (new_hires_enabled=true), то в список “Новые сотрудники” автоматически попадают:
все вновь созданные локально пользователи (админом),
все впервые появившиеся пользователи при интеграции/SSO/синхронизации из AD (модель D),
при этом фиксируется источник: LOCAL / AD.
FR-NH-6. Окно “новизны” и статус
У записи нового сотрудника должен быть статус, например:
Новый (ещё не выдавали задания)
Задания выданы
Завершено (по выбору админа или по факту 100% подтверждений — решение)
Рекомендуемое правило “новизны” (чтобы список не рос бесконечно):
пользователь считается “новым” до тех пор, пока администратор не снимет метку или не переведёт в “Завершено”.
4) Выдача заданий на ознакомление (ручной запуск админом)
FR-NH-7. Массовый выбор пользователей
В разделе “Новые сотрудники” администратор может:
выбрать одного / нескольких / всех пользователей (чекбоксы + “выбрать все”),
нажать действие «Отправить задания на ознакомление».
FR-NH-8. Состав задания (что включается)
При отправке заданий каждому выбранному пользователю назначаются все материалы, у которых:
material.new_hire_required = true (эффективный флаг на уровне материала),
материал опубликован,
и материал доступен пользователю по группе видимости (material.visibility_group_id).
Недоступные по группе материалы в задание не включаются (иначе будет “невыполнимое” задание).
FR-NH-9. Одно письмо пользователю на одну “выдачу”
После выдачи заданий пользователь получает одно письмо (TPL-NH-01):
заголовок “Вам назначены материалы для ознакомления”
список материалов (название + ссылка)
ссылка на страницу «Мои материалы для ознакомления»
пояснение: открыть каждый материал и нажать «Подтверждаю ознакомление»
Напоминаний нет. (как вы решили ранее)
FR-NH-10. Дедуп выдачи и повторные выдачи
Чтобы не плодить дубликаты:
назначение хранится на уровне user_id + material_id (уникально),
при повторном запуске для того же пользователя система:
добавляет только новые обязательные материалы,
по уже назначенным ничего не дублирует,
письмо можно отправлять снова, но с dedup key по событию (например NH-ASSIGN:<batch_id>).
5) Пользовательский интерфейс для “Мои ознакомления”
FR-NH-11. Единая страница “Мои материалы для ознакомления”
У пользователя должна быть страница (в профиле/главной):
список назначенных материалов (статус: ознакомился/не ознакомился),
фильтр “только не ознакомился”,
сортировка (по дате назначения, по названию, по критичности и т.п.),
переход в материал.
FR-NH-12. Подтверждение ознакомления — только кнопкой
В карточке материала, если есть активное назначение:
кнопка «Подтверждаю ознакомление с материалом»
фиксируем:
acknowledged_at (MSK),
acknowledged_version_id (текущая опубликованная версия на момент подтверждения)
6) Контроль админом и владельцем на карточке материала
FR-NH-13. Вкладка “Ознакомление” (как ранее) + раздел “Новые сотрудники”
Во вкладке материала «Ознакомление» добавить блок/таб:
«Новые сотрудники» — показывает, кому назначен этот материал в рамках “новых сотрудников”, и кто подтвердил.
FR-NH-14. Цветовая индикация статусов
По аналогии с ранее согласованным:
светло-зелёный — подтвердил ознакомление
светло-красный — не подтвердил и доступ есть
серый “Нет доступа сейчас” — пользователь назначен, но позже потерял доступ к материалу из-за смены группы видимости
